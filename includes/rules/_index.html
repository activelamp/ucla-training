Drupal automations

Rules module

The Rules module allows site administrators to define conditionally executed actions based on occurring events (known as reactive or ECA rules).

Event condition action

Excerpt from Content Intake Functional Outline

  • A notification email is generated when a user changes his/her password
  • A notification email is generated when a user account is disabled or enabled.
  • A notification email is generated when a Contributor account is assigned editing rights to content pages.
  • ...numerous more
  • You get the idea, whenever you say something happens when something happens, that's rules module

This is an engineering design pattern to loosely couple functionality

Rules module allows you to define new events, conditions, and actions.

A site administrator has the ability to connect functionality together through the UI

Rules is a subsystem framework within Drupal

You can extend rules

Rules allows you to make an extensible site for Administrators without any writing code

  • You can write all of your custom logic within the framework of Rules and let your admins control everything.

YMMV

Rules Module

    Pros:
  • Allows Site Admins to execute actions based on events
  • Extend functionality of the site thru the UI
    Cons:
  • Rules framework is very complex
  • Rules can be heavier than just writing code
  • The Drupal 8 version of Rules doesn't seem to be fully working

IMO... don't use Rules unless it is a requirement for your site admins to add events and actions

If you use Rules... understand the extra complexity Rules brings with it.

Rules has not hit stable release yet

I would use Rules for GCR

  • Help find bugs
  • Help fix bugs
  • Help release a stable version of Rules
