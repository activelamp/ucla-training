Access Control

Also known as Authorization

Authentication
vs
Authorization

  • Explain what the difference is

Drupal core comes with a thin layer of authorization capabilities

  • You can set what roles can create / edit specific node types
  • You can only have access to view all content or no content

Excerpt from Content Intake Functional Outline

A Contributor has a temporary account with permission to edit specifically assigned pages.
  • This isn't possible in Drupal core
  • We have to find a module, or write our own module

Drupal allows you to hook into access control

  • hook_node_access()
  • Grants - hook_node_access_records()

Out of scope for this training.

  • Drupal allows you to take complete control of access, however you want to achieve it.

GCR has two use cases for access control

  • The Contributor cannot revert changes once submitted for approval.
  • A Contributor has a temporary account with permission to edit specifically assigned pages.

Two different authorization models

  • Conflicting access - Grants have a priority that you can set, higher priority wins.

Contributed Modules

  • workflow_access - only Drupal 7
  • permissions_by_term - view only control
  • tac_lite - full control: read, update, delete

Exercise

  • Download tac_lite
  • Enable and configure based on the README
  • Manage access based on role
  • Manage access based on individual user
  • Edit the account of another user. Go to the tac_lite access tab under edit.
