Core

  • The official release of Drupal
  • Good starting point to build simple sites
  • You should not change any files in Drupal core
  • Follows the Open/Close principle

Exercise

  • Change the name of your Drupal site.
  • Add an article to your site
  • Edit the body field of your article

Modules

  • Extend core functionality with contribute code
  • Write code to make Drupal do what you want
  • Ability to enable and uninstall modules from the "Extend" menu in admin section.

We won't be covering module development today.

Exercise

  • Enable the "Settings Tray" module
  • Change the name of your Drupal site
  • Edit the User Account menu

Themes

  • Controls the appearance of your site
  • Uses Twig templates to define markup
  • Override markup that Drupal outputs by default
  • Change the appearance of your site with the "Appearance" menu in admin section.

Exercise

  • Download a contributed theme
  • Enable the theme

Download theme with Composer


# Download the theme from drupal packagist
$ composer require drupal/jethro
  • Show the new theme in the filesystem
  • Install the theme on the appearance tab

Profiles

  • Core, Modules, Themes, and Configuration packaged to create a "pre-baked" website
  • Site needs to be installed initially with profile
  • The basis for Drupal distributions

Site Farm Seed is a base distribution that we will extend

Exercise

  • Download the Acquia Lightning distribution
  • Install Drupal using Lightning profile

Download distribution with Composer


# Download the distro from drupal packagist
$ composer create-project acquia/lightning-project lightning
$ cd lightning
$ mkdir .docksal
$ fin up
$ fin drush si --db-url=mysql://user:user@db/default --root=/var/www/docroot
$ open http://lightning.docksal
