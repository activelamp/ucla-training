General Concepts (cont)

Content objects that come in core.

  • Node (Content)
  • Comment
  • Taxonomy
  • User

Node

A node is the generic term for any piece of content on your web site. The content type of the node will define what fields (or specific information) that node displays.

Structured Content

Do not think of your site in terms of pages, think in terms of content.

Content can take many shapes... (Home Page, Sidebar, RSS, etc...)

  • Talk about the importance of having a content model
  • Form follows function, you can build functionality first
  • Editors can begin getting used to entering content before frontend is done

Comment

Hint: Use a third party service like Disqus for comments

Taxonomy

The practice of classifying content... categories, tags, etc...

  • Sections of sites
  • RSS Feeds
  • Menus
  • Access Control
  • Lots of modules based around Taxonomy

User

Users that log into your site

  • Authentication
  • Roles
  • Permissions

Entity Types

Create custom schemas that don't fit a Core entity.

  • ...if you need to create custom tables
  • ...if you want to define a PHP Class
  • ...if the data is not content

All Entities in Drupal are fieldable

You can create any "properties" on content objects all through the Drupal UI

Regions & Blocks

Pages on your Drupal site are laid out in Regions. Blocks are discrete chunks of information that are displayed in the regions of your site's pages.

Views

Visual query build for creating lists of content

Path

  • Internal Drupal path
  • Path Alias

Image styles

Image styles are used to set presets for image processing. Define effects in the Drupal UI to apply to uploaded images

  • Scale & Crop
  • Desaturate
  • Rotate
  • etc...

Permissions

Permissions can be set to be very specific and granule making for a powerful feature to use when developing a site's structure.

Apply permissions to roles

  • Administrator
  • Content Editor
  • Authenticated
  • Anonymous

Configuration

Configuration is the collection of admin settings that determine how the site functions, as opposed to the content of the site.
